# una-ihcux-lista03

 - Explicação da 1ª Heurística de Nielsen
O código apresentado aplica a 1ª Heurística de Nielsen (Visibilidade do Status do Sistema), que preconiza que o sistema deve manter o usuário permanentemente informado sobre o seu estado atual por meio de feedback adequado e tempestivo.

<img width="1366" height="768" alt="minha-evidencia" src="https://github.com/user-attachments/assets/4c84c3ff-b8ae-407c-8b24-6696625ddd08" />


Pergunta de Reflexão (IHC): > "Se o programa demorasse 10 segundos para finalizar, mas ficasse com a tela totalmente parada (em branco), sem as mensagens de '[PROCESSANDO]', o que o usuário provavelmente pensaria? Como a visibilidade de status melhora a Experiência do Usuário (UX)?"

Pensamento do usuario: O site esta lento, não esta funcionando, minha internet esta lenta

Como a visibilidade de status melhora a Experiência do Usuário (UX)?
Não deixa o usuario esperando por alguma atualização na interface, tirando a sensação de perca e tempo.
